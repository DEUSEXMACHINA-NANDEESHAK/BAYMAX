Why Blockchain Is Not Used

The system relies on Vertex 2.0’s DAG-based, leaderless BFT consensus engine, not blockchain.
Blockchain’s high latency, reliance on internet, and single points of failure make it unsuitable.
Vertex 2.0 offers sub-100ms finality, offline capability, and Byzantine fault tolerance, ideal for machine coordination.
The system provides cryptographic identity, tamper-proof event ordering, and verifiable proofs without blockchain’s drawbacks.

Use of DAG in Vertex 2.0 for Fast Consensus
Vertex 2.0 employs a Directed Acyclic Graph (DAG) structure allowing agents to write events simultaneously, enabling sub-100ms consensus without voting rounds or leaders.

DAG replaces traditional blockchain's sequential blocks, allowing concurrent event writing.
Multiple agents (e.g., Agent A, B, C) publish events simultaneously.
The system derives the correct event order mathematically from the DAG.
Achieves fast, leaderless, Byzantine Fault Tolerant (BFT) consensus. ​

MQTT Protocol for Agent Communication
MQTT is a lightweight messaging protocol used for real-time communication among agents, similar to a radio channel.

Agents publish messages to topics (e.g., "swarm /state/drone-01"). ​
Agents subscribe to topics to receive messages instantly. ​
Example: Drone-01 publishes state; Rover-03 subscribes to all agents' states. ​
Implementation in TypeScript with mqtt.js is straightforward (2 lines).

FoxMQ as a Decentralized MQTT Broker
FoxMQ replaces central MQTT brokers with a 4-node cluster using Vertex 2.0 BFT consensus, ensuring high availability. ​

Agents connect to local FoxMQ nodes, which communicate via BFT.
The cluster tolerates node failures (e.g., 1 node down still operational).
No Rust coding required; run pre-built binaries and connect via MQTT in TypeScript.
Ensures decentralized, fault-tolerant message passing.

CRDTs for Shared State Management
CRDTs enable conflict-free, concurrent updates to shared data like maps, ensuring consistency without conflicts. ​

Example: Multiple agents update a shared grid map simultaneously. ​
Merging updates is conflict-free, similar to Google Docs.
Libraries like yjs or automerge in TypeScript facilitate CRDT implementation.
Supports real-time, consistent shared state across agents. ​
